基于 hugo 在 GitHub 搭建个人博客

本文记录的是基于 hugo 的 GitHub 博客的搭建过程

hugo 装载

  1. 在hugo官方的 GitHub 站点中，按照 tags 下载新版本的hugo工具压缩包，本文基于的是 Windows 系统。使用下面的命令创建一个名为dev的博客根目录：
1

hugo new site dev
  1. 在hugo中的主题模板页选择自己的博客模板，本文选择的是stack模板，将安装包解压到\theme目录下，将包名中的版本号去掉。
  2. 将主题包中\exampleSite中的hugo.yaml文件和content目录复制到dev目录下。

运行hugo博客：hugo server -D

配置自动化部署

创建一个用于自动化部署的仓库，取名为hugo-main。

在dev目录下创建.gitignore，避免上传下列这些文件目录：

  • public
  • resources
  • hugo_build.lock
  • hugo.exe

然后，创建dev\.github\workflows\deploy.yaml：

name: deploy

# 代码提交到main分支时触发github action
on:
  push:
    branches:
      - main

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v4
        with:
          fetch-depth: 0

      - name: Setup Hugo
        uses: peaceiris/actions-hugo@v3
        with:
          hugo-version: "latest"
          extended: true

      - name: Build Web
        run: hugo -D

      - name: Deploy Web
        uses: peaceiris/actions-gh-pages@v4
        with:
          PERSONAL_TOKEN: ${{ secrets.TOKEN }}
          EXTERNAL_REPOSITORY: AlanKSorata/alanksorata.github.io
          PUBLISH_BRANCH: main
          PUBLISH_DIR: ./public
          commit_message: auto deploy
  • PERSONAL_TOKEN处配置TOKEN需要在hugo-main中填入在个人设置中生成的token。
  • EXTERNAL_REPOSITORY处按照GitHub名/网站仓库名填写。

上传到 hugo-main

在dev下配置git并上传到hugo-main仓库中，上传后可见其自动化部署的Action
